# Генератор постов для товаров

Flask-сайт для создания уникальных продающих постов с помощью языковой модели (GPT) через ProxyAPI.

## Возможности

- **Генерация постов с ИИ** — GPT-модель создаёт оригинальные посты
- **4 тона текста** — дружелюбный, восторженный, деловой, спокойный
- **Готовые хэштеги и CTA** — модель добавляет релевантные теги и призыв к действию
- **Копирование в один клик** — готовый пост копируется в буфер обмена
- **Редактирование поста** — редактируйте текст перед публикацией
- **Подсчёт символов** — видно количество символов в посте
- **Проверка орфографии** — проверяйте текст на ошибки
- **Предпросмотр** — посмотрите как пост будет выглядеть
- **Публикация в ВК** — опубликуйте пост или запланируйте на время
- **Избранное** — сохраняйте посты для последующего использования
- **Отложенные посты** — запланируйте публикацию на нужное время
- **Без базы данных** — простой код для понимания

## БЕЗОПАСНОСТЬ

### Все ключи хранятся в переменных окружения

Это сделано для защиты секретных данных от утечки в GitHub.

#### Файлы исключённые из Git (.gitignore):

| Файл | Почему исключён |
|------|-----------------|
| `config.py` | Может содержать ключи |
| `run.bat` | Локальный запуск с ключами |
| `.env*` | Переменные окружения |
| `VK_ACCESS_TOKEN*` | Токен ВКонтакте |
| `VK_GROUP_ID*` | ID группы ВК |
| `PROXY_API_KEY*` | Ключ ProxyAPI |
| `favorites.json` | Данные пользователя |
| `scheduled_posts.json` | Данные пользователя |
| `__pycache__/` | Скомпилированный код |

#### Как работает защита:

1. `config.py` — читает ключи из `os.environ.get()`
2. `run.bat` — устанавливает переменные перед запуском
3. Ключи **НЕ сохраняются** в коде проекта
4. При пуше в Git — секреты не попадают

### Для других пользователей:

1. Скопируйте `run.bat.example` → `run.bat`
2. Вставьте свои ключи в `run.bat`
3. Или экспортируйте переменные окружения вручную

## Установка и запуск

### 1. Установите зависимости

```bash
pip install -r requirements.txt
```

### 2. Настройте ключи

Отредактируйте файл `run.bat`:

```batch
set PROXY_API_KEY=ваш-прокси-ключ
set VK_ACCESS_TOKEN=ваш-токен-вк
set VK_GROUP_ID=123456789
```

### 3. Запустите сервер

**Windows:**
```bash
run.bat
```

**Linux/Mac:**
```bash
export PROXY_API_KEY=ваш-прокси-ключ
export VK_ACCESS_TOKEN=ваш-токен-вк
export VK_GROUP_ID=123456789
python app.py
```

### 4. Откройте в браузере

```
http://127.0.0.1:5004
```

## ИНТЕРФЕЙС

### Вкладка "Генератор"

1. Заполните форму:
   - Название товара
   - Описание* (обязательно)
   - Цена/выгода
   - Тон текста (4 варианта)

2. Нажмите "Сформировать пост"

3. Под постом доступно:
   - **Скопировать** — копирует в буфер обмена
   - **В избранное** — сохраняет пост
   - **Редактировать** ✏️ — открывает редактор

4. Блок "Публикация во ВКонтакте":
   - Статус подключения ВК
   - "Опубликовать сейчас" — публикует сразу
   - "Запланировать" — выберите дату и время

5. Блок "Предпросмотр":
   - Показывает как пост будет выглядеть

### Вкладка "Избранное"

- Список сохранённых постов
- "Использовать" — загружает пост в генератор
- "Удалить" — удаляет из избранного
- "Обновить" — перезагружает список

### Вкладка "Отложенные"

- Список запланированных постов
- "Удалить" — отменяет публикацию
- "Обновить" — перезагружает список

## РЕДАКТИРОВАНИЕ ПОСТА

1. Нажмите кнопку "Редактировать" под постом
2. Откроется окно редактора:
   - Поле для изменения текста
   - Счётчик символов
   - Кнопка "Проверить орфографию"
3. Внесите изменения
4. Нажмите "Сохранить"
5. Пост обновится и его можно опубликовать

### Проверка орфографии

- Проверяет распространённые ошибки
- Ищет повторяющиеся буквы
- Выявляет слитное написание слов
- Даёт рекомендации по исправлению

## ДЕПЛОЙ НА PYTHONANYWHERE

### Шаг 1 — Локально: убедитесь что config.py не в Git

```bash
git ls-files | findstr config.py
```

Если выводит `config.py` — удалите из отслеживания:
```bash
git rm --cached config.py
git commit -m "Удалён config.py из репозитория"
```

### Шаг 2 — На PythonAnywhere: Загрузите файлы

Имя папки: `SiteGenPostSimp`

**Загрузить:**
```
app.py
api_client.py
vk_client.py
favorites.py
scheduled.py
config.py.example
templates/index.html
static/style.css
requirements.txt
```

**НЕ загружать:**
- `config.py` (там ваши ключи!)
- `.gitignore`
- `favorites.json`
- `scheduled_posts.json`
- `run.bat`

### Шаг 3 — На PythonAnywhere: Создайте config.py

В папке `SiteGenPostSimp` создайте файл `config.py`:
```python
import os

PROXY_API_KEY = ""  # Вставьте ключ сюда
PROXY_API_BASE_URL = "https://api.proxyapi.ru"
MODEL_NAME = "gpt-5.4-mini"
MAX_TOKENS = 800
MAX_TOKENS_PARAM = "max_completion_tokens"
TEMPERATURE = 0.9
API_TIMEOUT = 60

VK_ACCESS_TOKEN = ""  # Вставьте токен сюда
VK_GROUP_ID = ""      # Вставьте ID группы сюда

BASE_DIR = os.path.dirname(os.path.abspath(__file__))
FAVORITES_FILE = os.path.join(BASE_DIR, "favorites.json")
SCHEDULED_FILE = os.path.join(BASE_DIR, "scheduled_posts.json")
```

### Шаг 4 — Настройка Web-приложения

1. **Web** → **Add a new web app** → **Manual configuration** → Python 3.x
2. **WSGI file** → редактируйте, замените на:
```python
import sys
sys.path.insert(0, '/home/ВАШ-USERNAME/SiteGenPostSimp')

from app import app as application
```
3. **Static files** → добавить:
   - URL: `/static/` → Directory: `/home/ВАШ-USERNAME/SiteGenPostSimp/static/`

### Шаг 5: Установите зависимости

**Bash console:**
```bash
cd SiteGenPostSimp
pip install flask requests
```

### Шаг 6: Перезагрузите и проверьте

**Web** → **Reload** → откройте ваш сайт.
