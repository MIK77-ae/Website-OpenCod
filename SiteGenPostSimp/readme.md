# Генератор постов для товаров

Flask-сайт для создания уникальных продающих постов с помощью языковой модели (GPT) через ProxyAPI.

## Возможности

- **Генерация постов с ИИ** — GPT-модель создаёт оригинальные посты
- **4 тона текста** — дружелюбный, восторженный, деловой, спокойный
- **Готовые хэштеги и CTA** — модель добавляет релевантные теги и призыв к действию
- **Копирование в один клик** — готовый пост копируется в буфер обмена
- **Редактирование поста** — редактируйте текст перед публикацией
- **Подсчёт символов** — видно количество символов в посте
- **Проверка орфографии** — проверяйте текст на ошибки
- **Предпросмотр** — посмотрите как пост будет выглядеть в ВК
- **Публикация в ВК** — опубликуйте пост или запланируйте на время
- **Избранное** — сохраняйте посты для последующего использования
- **Отложенные посты** — запланируйте публикацию на нужное время
- **Без базы данных** — простой код для понимания

# Установка и запуск (локально)

## 1. Установите зависимости

```bash
pip install -r requirements.txt
```

## 2. Настройте ключи

Создайте файл `.env` в корне проекта:

```env
PROXY_API_KEY=ваш-прокси-ключ
VK_ACCESS_TOKEN=ваш-токен-вк
VK_GROUP_ID=123456789
```

Или через командную строку (Windows):
```batch
set PROXY_API_KEY=ваш-ключ
set VK_ACCESS_TOKEN=ваш-токен
set VK_GROUP_ID=123456789
```

## 3. Запустите сервер

```bash
python app.py
```

## 4. Откройте в браузере

```
http://127.0.0.1:5004
```

## Частые вопросы

**Нужен ли VPN для работы сайта на PythonAnywhere?**

Нет. PythonAnywhere имеет полный доступ к интернету и может обращаться к `api.proxyapi.ru` и `api.vk.com` напрямую. VPN нужен только если API заблокированы в России.

**Безопасно ли держать ключи в config.py на PythonAnywhere?**

Да. Файлы на PythonAnywhere **не публичные** — только вы имеете к ним доступ. Это гораздо безопаснее чем GitHub, где любой может увидеть ваш код.

**А ключи не попадут в GitHub?**

Нет. Мы уже удалили `config.py` из Git:
```bash
git rm --cached config.py
git commit -m "Удалён config.py из репозитория"
git push
```


## Шаг 3 — Создайте config.py на сервере

В папке `SiteGenPostSimp` создайте файл `config.py`:

```python
import os

PROXY_API_KEY = "ваш-прокси-ключ"
PROXY_API_BASE_URL = "https://api.proxyapi.ru"
MODEL_NAME = "gpt-5.4-mini"
MAX_TOKENS = 800
MAX_TOKENS_PARAM = "max_completion_tokens"
TEMPERATURE = 0.9
API_TIMEOUT = 60

VK_ACCESS_TOKEN = "ваш-токен-вк"
VK_GROUP_ID = "238535077"

BASE_DIR = os.path.dirname(os.path.abspath(__file__))
FAVORITES_FILE = os.path.join(BASE_DIR, "favorites.json")
SCHEDULED_FILE = os.path.join(BASE_DIR, "scheduled_posts.json")
```

## Шаг 4 — Настройте WSGI (Flask preset)

Если при создании выбрали **Flask**, WSGI уже настроен частично.

Найдите: **Web** → **WSGI configuration file** → **Edit**

Замените путь на ваш (замените `ВАШ-ЮЗЕРНЕЙМ` и `mysite`):

**Было:**
```python
project_home = '/home/ВАШ-ЮЗЕРНЕЙМ/mysite'
```

**Стало:**
```python
project_home = '/home/ВАШ-ЮЗЕРНЕЙМ/SiteGenPostSimp'
```

### Что такое WSGI?

WSGI (Web Server Gateway Interface) — это мост между веб-сервером (Apache) и вашим Flask-приложением. Без него сервер не знает как запустить сайт.

## Решение проблем

**Ошибка 401 Unauthorized (ключ не работает):**
- Проверьте что в `config.py` на PythonAnywhere ключ заполнен
- Ключ должен быть в кавычках: `PROXY_API_KEY = "ваш-ключ"`
- После изменения: **Web** → **Reload**

**Ошибка 404:** Проверьте что путь в WSGI правильный.

**Ошибка 500:** Откройте **error log** (на странице Web) — там будет описание ошибки.

**Модуль не найден:** Установите зависимости заново:
```bash
pip install --user flask requests
```

## БЕЗОПАСНОСТЬ

### Файлы исключённые из Git

| Файл | Почему |
|------|--------|
| `config.py` | Содержит ключи API |
| `.env*` | Переменные окружения |
| `run.bat` | Локальный запуск с ключами |
| `favorites.json` | Данные пользователя |
| `scheduled_posts.json` | Данные пользователя |

### Как работает защита

1. `config.py` читает ключи из переменных окружения `os.environ.get()`
2. Ключи **НЕ сохраняются** в коде
3. При пуше в Git секреты не попадают
